# CLAUDE.md

Guidance for working in this repo. Read this first.

## Project

**yearly** (codename **Kaizen**) — a local-first, minimalist personal goal/habit tracker for iOS & Android. Single user, offline, no accounts. A goal is either a **checkbox** (yes/no) or a **numeric** value with a custom unit, tracked over a period (daily → yearly) and judged by a per-goal **aggregation × comparator** rule. Design language: near-monochrome, Vercel-neutral palette, Geist + Space Mono, Fold-app-inspired layout (grey canvas, white cards, floating pill nav).

Product/research background lives in `docs/plans/`.

## Stack

Expo SDK 55 · React Native 0.83 · React 19 · expo-router (file-based, typed routes) · TypeScript strict · expo-sqlite · Zustand · react-native-reanimated v4 · react-native-svg · expo-notifications · expo-haptics · @expo-google-fonts (Geist, Space Mono). New Architecture (Fabric/TurboModules/bridgeless) is always on.

## Commands

```bash
npx expo start --ios          # run on iOS simulator via Expo Go (all native deps used are in Expo Go)
npx tsc --noEmit              # typecheck — must be clean before committing
npx prettier --write "src/**/*.{ts,tsx}"   # format (tabs, single quotes)

# Android APK (release, debug-signed → installs directly):
npx expo prebuild --platform android --clean --no-install   # only needed after adding NATIVE deps
cd android && ./gradlew assembleRelease
# Output: android/app/build/outputs/apk/release/app-release.apk
```

**Build gotchas**
- The build MUST use **JDK 17**. `/usr/libexec/java_home -v 17` is unreliable (returns 25). Set explicitly:
  `export JAVA_HOME=/opt/homebrew/opt/openjdk@17/libexec/openjdk.jdk/Contents/Home` and `export ANDROID_HOME=$HOME/Library/Android/sdk`. Under JDK 25 the CMake configure tasks fail.
- `ios/` and `android/` are **gitignored (CNG)** — they're regenerated by prebuild. Don't commit them. If only JS changed, you can skip prebuild; `gradlew assembleRelease` re-bundles JS.
- `SEED_DEV` flag in `src/core/store/useStore.ts` seeds sample goals/entries when the DB is empty — keep it **false** for release builds.

## Architecture

```
src/
  core/                  # shared, cross-feature
    theme/               # design tokens (colors, spacing, radii, fontSize, fontFamily, shadow) + fonts loader
    ui/                  # one component per file: Text, Button, Card, IconButton, IconBadge, Glyph,
                         #   Pill, SegmentedControl, ScreenHeader, StatTile, Row, Divider, ProgressRing,
                         #   Sparkline, AnimatedNumber, NotificationBanner — re-exported via index.ts barrel
    domain/              # pure logic: types.ts, period.ts, aggregation.ts, format.ts (NO React)
    db/                  # expo-sqlite data layer: index.ts (queries), seed.ts (dev only)
    store/               # useStore.ts (Zustand) — loads goals/entries/meta into memory, exposes CRUD
    lib/                 # haptics.ts, notifications.ts
  features/
    goals/{components,screens,types}
    calendar/{components,screens}
  app/                   # expo-router ROUTES ONLY — thin files re-exporting feature screens + layouts
```

- Path alias: `@/*` → `src/*`.
- `app/` route files are one-liners: `export { default } from '@/features/.../x-screen'`. Layouts (`_layout.tsx`) hold providers/nav config; root layout does font load + `store.init()` + notification permission + the permission banner.
- Data flow: **only `Goal` + `Entry` are persisted** (sqlite). Everything else (completion %, streaks, period rollups, the calendar heatmap) is **derived** by `core/domain/aggregation.ts`. The Zustand store holds all goals/entries in memory (data is tiny) and re-selects after mutations.

### Key domain model
- `Goal`: type (checkbox|numeric), unit, period (daily|weekly|monthly|quarterly|semiannual|yearly), `aggregation` (sum|mean|median|mode|min|max|count) × `comparator` (at_least|at_most), targetValue, icon, color, `reminders: {hour,minute}[]`.
- Success of a period = `aggregate(entries in calendar-aligned window) compared to target`. `mean` uses elapsed-days as denominator (missing day = 0). Streaks count **met periods**. `dayCompletion` (0..1 per day) drives the Today ratio + heatmap shade; `evaluatePeriod` drives detail streaks/progress.
- Reminders are stored on the goal and (re)scheduled as daily local notifications on grant and on any goal change (`core/lib/notifications.ts`).

### Design system (`core/theme`)
- **Monochrome, Vercel-neutral**: `bg` grey canvas, `surface` white cards, `ink` `#000`, neutral grays. Per-goal accent colors are the only color, used to tint goal icon badges. No green/blue chrome.
- **Type**: Geist (sans) for prose/headings via the `Text` component; **Space Mono for numbers and small uppercase labels** (`Text` `mono` prop; `caption` defaults to mono). Sizes/weights are tokens — never hardcode.
- **Numbers**: full values use comma grouping (`formatCount`/`formatPrecise`); compact `K` (`formatValue`) only in space-constrained calendar cells.
- **Motion**: reanimated v4, timing with `Easing.out(Easing.cubic)`/bezier (no springy bounce). Tasteful haptics via `core/lib/haptics` on toggles/steps/selection.
- **Shadows**: use the cross-platform **`boxShadow`** style prop (RN 0.76+), centralized in `theme.shadow.card`/`theme.shadow.float` — spread those, don't hand-roll. **Never use Android `elevation`** or the legacy `shadowColor`/`shadowOpacity`/`shadowRadius`/`shadowOffset` props: `elevation` renders a boxy artifact under animated/parent opacity (card entrance fades, dimmed cards) on Android. `boxShadow` renders correctly on both platforms under opacity.
- **Text inputs**: set `selectionColor` to a translucent ink (e.g. `rgba(10,10,10,0.25)`) and `cursorColor` to ink — a solid selection color hides the text on Android.

## Conventions
- TypeScript strict; **no `any`** (use `unknown` + narrowing). Type props with named `interface`/`type` — **no inline prop object types** for exported components. Avoid `React.FC`.
- **One exported component per file.** Tightly-coupled private subcomponents may share the file. `core/ui` files are PascalCase; feature files are kebab-case.
- Prettier: tabs, single quotes, jsx single quotes, trailing commas. Match surrounding style.
- Always render text via the `Text` component (never RN `Text`) so the font system applies.
- Keep `app/` for routes only. Put logic in features/core.

## Modern RN/Expo best practices (2025–2026)

- **expo-router**: keep `experiments.typedRoutes` on; thin route files; `(group)` for layout grouping; `useLocalSearchParams<{...}>()`; build hrefs as `{ pathname, params }`. Tests/components never live in `app/`.
- **State**: local `useState` first; Zustand for global/domain state with **selectors** (`useStore(s => s.x)`, `useShallow` for multi-field) — never subscribe to the whole store. Keep state flat; **derive, don't store**.
- **Components**: function components; composition over boolean-flag explosions; controlled inputs for validated fields; add accessibility props (`accessibilityRole`/`accessibilityLabel`) + adequate hit targets.
- **Memoization**: React 19 + React Compiler auto-memoizes — don't add defensive `useMemo`/`useCallback`; keep them only where referential identity matters (list item props, gesture/scroll handlers, identity-comparing libs).
- **Performance**: FlashList v2 (not FlatList) for long lists; no inline funcs/objects/styles in list items or animated components; expo-image with `contentFit` + cache policy for images; keep animation on the UI thread.
- **Reanimated**: read/write `sharedValue.value` only in worklets / `useAnimatedStyle` / callbacks — never during JS render; use `runOnJS` to call back into React. Prefer CSS-style transitions for simple non-gesture motion.
- **Styling**: `StyleSheet`/tokens over scattered inline styles (dynamic values excepted); theme via tokens; `react-native-safe-area-context` for insets (never hardcode); `useColorScheme()` read each render if doing dark mode.
- **Data**: keep SQL in `core/db` (repositories) — UI never issues SQL; version migrations with `PRAGMA user_version` / idempotent `ALTER`s in a transaction; use transactions for multi-writes.
- **Native/build**: add native capability via config plugins; CNG (gitignored `ios`/`android`); dev build (`expo-dev-client`) or Expo Go for iteration; request notification permission with a soft in-app prompt (the banner) — don't burn the one-time iOS dialog; create the Android notification channel before sending.

### Pitfalls
- Non-route files inside `app/` (breaks expo-router).
- Subscribing to the whole Zustand store → app-wide re-renders.
- Reading `sharedValue.value` in render; `setState` in a worklet without `runOnJS`.
- Storing derived data instead of computing it.
- Committing `ios/`/`android/` (CNG conflict); building under the wrong JDK (use 17).
- Using Android `elevation` for shadows — use `boxShadow` (the elevation layer boxes out under opacity animations/dimming).
